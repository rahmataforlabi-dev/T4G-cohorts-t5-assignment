##Q4.How does a Firewall work?
A firewall acts like a security barrier between a network and the internet. It monitors incoming and outgoing traffic, allowing safe connections while blocking unauthorized or potentially harmful access. Its main purpose is to protect systems from cyber threats.
How does a Load Balancer work?

     A load balancer manages incoming network traffic by distributing user requests across multiple servers. This prevents any single server from becoming overloaded and helps ensure that applications remain fast and available, even during periods of high traffic.

    Difference between a Firewall and a Load Balancer
A firewall protects the network, while a load balancer helps the network handle many users efficiently.

##Q5. Web Server vs. Application Server.
​Web Server: Handles static files like, raw HTML, CSS, and image.
​Application Server: Handles the heavy lifting logic, runs backend code, and talks to the database.
​ A backend developer works mostly with the Application Server because that is where the actual programming logic, user authentication, and business rules live.

##Q6.​ A client does not connect directly to the database because it would create serious security risks. If users could access the database directly, they might be able to view, or delete sensitive information without permission. Instead, requests are sent through an application server, which verifies the user's identity, checks their login credentials, and ensures they can only access the data they are authorized to see. This approach helps protect the database and keeps user information secure.
